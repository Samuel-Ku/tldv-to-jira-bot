# TL;DV Pipeline 🤖

Telegram бот для автоматичного аналізу зустрічей з tl;dv. Знаходить action items за допомогою AI та ключових слів, надсилає фрагменти з таймстемпами.

## Архітектура (спрощена)

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   Telegram  │───▶│ API tldv.io │───▶│  LLM аналіз │───▶│   Telegram  │
│  (посилання)│    │(транскрипт) │    │(завдання/AI)│    │ (результати)│
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                                              │
                                              ▼
                                       ┌─────────────┐
                                       │ Keyword scan│
                                       │  (fallback) │
                                       └─────────────┘
```

Бот працює безпосередньо з готовими транскриптами від tl;dv — не потрібно завантажувати відео або транскрибувати аудіо локально. AI аналіз виділяє задачі автоматично.

## Функціонал

### 🔍 Два режими аналізу

1. **AI Аналіз (LLM)** — інтелектуальний аналіз транскрипту:
   - Автоматичне виділення action items
   - Визначення пріоритетів (high/medium/low)
   - Ідентифікація виконавців
   - Генерація summary зустрічі
   - Ключові моменти

2. **Keyword Matching** — пошук за ключовими словами:
   - Резервний метод якщо LLM недоступний
   - Кастомізований список ключових слів
   - Контекст ±2 сегменти навколо збігу

3. **Інтеграція з Jira** — автоматичне створення задач:
   - Створення задач безпосередньо в Jira з Telegram
   - Автоматичне заповнення виконавців (якщо вони співпадають з юзерами в Jira)
   - Прикріплення оригінальних таймстемпів до тасок

## Швидкий старт

### 1. Системні залежності

```bash
# Kali Linux / Debian / Ubuntu
sudo apt update
sudo apt install -y python3 python3-venv python3-pip

# Перевірка Python версії (потрібна 3.9+)
python3 --version
```

### 2. Python оточення

```bash
cd ~/tldv-pipeline

# Створення віртуального оточення
python3 -m venv .venv
source .venv/bin/activate

# Встановлення залежностей
pip install -r requirements.txt
```

### 3. Налаштування змінних оточення

```bash
cp .env.example .env
nano .env  # відредагуйте
```

Обов'язково заповніть:
- `TELEGRAM_BOT_TOKEN` — отримайте у [@BotFather](https://t.me/botfather)
- `TELEGRAM_ALLOWED_IDS` — ваш Telegram ID (дізнатись у [@userinfobot](https://t.me/userinfobot))

### 4. Запуск бота

```bash
source .venv/bin/activate
python bot.py
```

Бот запуститься і почне слухати повідомлення.

## 🚀 Інструкція по використанню

### 1. Отримайте доступ до бота

1. Знайдіть бота в Telegram за його ім'ям (наприклад, `@your_bot_name`)
2. Натисніть кнопку **Start** або надішліть команду `/start`
3. Якщо ваш Telegram ID не в списку дозволених — зверніться до адміністратора

### 2. Надішліть посилання на зустріч

Бот підтримує різні формати посилань tl;dv:
- `https://tldv.io/app/meetings/XXXXXXXX`
- `https://tldv.io/share/XXXXXXXX`
- `https://app.tldv.io/share/XXXXXXXX`

**Що відбувається далі:**
1. 🔍 Бот отримує транскрипт з tl;dv API
2. 🤖 LLM аналізує текст та виділяє action items
3. 📋 Результати групуються за учасниками
4. ⏱️ Додаються таймстемпи до кожної задачі
5. 📤 Отримуєте структурований звіт у чат

### 3. Керування ключовими словами (опціонально)

**Переглянути список:**
```
/keywords
```

**Додати нове ключове слово:**
```
/add треба зробити
/add створити задачу
```

**Видалити ключове слово:**
```
/remove треба зробити
```

> 💡 Ключові слова використовуються як резервний метод (fallback) якщо LLM аналіз недоступний.

### 4. Приклади використання

**Просто надішліть посилання:**
```
https://tldv.io/app/meetings/abc123def456
```

**Після обробки бот надішле:**
- 📋 Summary зустрічі
- 📝 Список задач згрупованих по людях
- ⏱️ Таймстемпи для кожної задачі
- 🔴 Пріоритети (high/medium/low)

**Якщо потрібна допомога:**
```
/help
```

## Ключові слова

За замовчуванням налаштовані слова для пошуку action items:
- `треба зробити`, `потрібно зробити`
- `створити таск`, `створити задачу`, `створити тікет`
- `завести баг`, `завести тікет`
- `джира`, `jira`, `бэклог`
- `хто візьме`, `хто бере`, `візьми на себе`
- `дедлайн`, `deadline`, `ASAP`
- `follow up`, `next step`
- українські та російські варіанти

Додавайте свої через `/add <фраза>` команду або редагуйте `keywords.json`.

## Опис роботи

1. **Отримання транскрипту** — `transcript_fetcher.py` запитує готовий транскрипт з API tldv.io (`/transcript` або `/watch-page` endpoints)
2. **Парсинг** — підтримка двох форматів відповіді (складний word-level та простий phrase-level)
3. **AI Аналіз** — `llm_analyzer.py` аналізує транскрипт через LLM:
   - Виділяє action items з пріоритетами
   - Ідентифікує виконавців
   - Генерує summary зустрічі
4. **Fallback Аналіз** — якщо LLM недоступний, використовується `extractor.py` з keyword matching
5. **Вивід** — Telegram бот надсилає результати з таймстемпами [MM:SS]
